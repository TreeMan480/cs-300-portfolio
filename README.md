## Reflection  

**What was the problem you were solving in this course?**  
The main problem I was solving in this course was figuring out how to store, organize, and access data efficiently. Specifically, I worked on sorting and printing a list of computer science courses in alphanumeric order. This required choosing the right data structure and making sure the program could handle the data correctly.

**How did you approach the problem?**  
I approached the problem by breaking it down into smaller steps and comparing different data structures like vectors, hash tables, and binary search trees. I focused on how each one performed in terms of run-time and memory usage. Understanding data structures is important because the wrong choice can make a program slower or harder to manage, especially with larger datasets.

**How did you overcome any roadblocks?**  
One of the biggest challenges I ran into was getting everything to work correctly when reading data from files and organizing it. I had to debug issues step by step and sometimes go back and rethink my logic. I also used trial and error along with testing small parts of the program to figure out what was going wrong.

**How has this project expanded your approach to designing software?**  
This project helped me realize that planning is really important before writing code. Instead of jumping straight into coding, I started thinking more about structure and efficiency first. It changed the way I approach problems because I now consider performance and scalability early on.

**How has your programming evolved in terms of maintainability and readability?**  
Over time, my code has become more organized and easier to read. I started using better naming conventions and breaking my code into smaller, more manageable sections. This makes it easier to debug and update later. I also understand now how important it is to write code that someone else can read and understand.
